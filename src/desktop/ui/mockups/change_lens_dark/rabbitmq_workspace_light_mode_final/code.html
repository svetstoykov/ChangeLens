lightbulb Area explanation

How this area works

The new PaymentResultConsumer acts as the perimeter defense for async payment callbacks. It validates structural integrity before allowing state transitions in the core domain.

Inferred Architecture

Security Enforcement

Authentication logic has been pushed to the message boundary. The consumer explicitly verifies the X-Signature header linkPaymentResultConsumer.cs:42.

This aligns with the 'Zero Trust Internal Network' policy observed in previous PRs in the ExamplePaymentService area.

warning Potential Dead-Letter Leak

When UnauthenticatedMessageException is thrown, Masstransit defaults to moving the message to an error queue. However, the external RabbitMQ topology config (in appsettings.json) must define the DLX binding to prevent message loss.

Exception Modernization

Replaced generic Exception with domain-specific PaymentNotFoundException, improving telemetry categorization in Datadog.
